# CLAUDE.md — schwung-midiverb

Schwung audio-FX module emulating Alesis Midiverb / Midifex / Midiverb II.

## Architecture

- `src/dsp/plugin.c` — `audio_fx_api_v2` entry, params, process_block, resampler glue
- `src/dsp/midiverb_core.{c,h}` — per-instance state, unit/program dispatch
- `src/dsp/resampler.{c,h}` + `resampler_coeffs.h` — 32-tap polyphase windowed-sinc, 44.1 ↔ 23.4 kHz, 256 phases
- `src/dsp/decompiled-*.h`, `names-*.h` — vendored upstream (GPL-3)

## Audio path

```
Move int16 stereo @ 44.1 kHz
  → float
  → downsample 44.1 → 23.4 (per channel)
  → mono = (L + R)/2 * input_gain + feedback
  → HPF (one-pole)
  → 13-bit int16 (downshift by 3)
  → effect_fn (decompiled DASP DSP)
  → 13-bit int16 stereo
  → 16-bit float (upshift by 3, clip)
  → LPF (one-pole)
  → M/S width
  → upsample 23.4 → 44.1 (per channel)
  → wet * mix + dry * (1-mix), output_gain, clip
  → int16 out
```

Critical: DASP is **13-bit at 23.4 kHz**, not 16-bit. Input must be downshifted
by 3 before `effect_fn`, output upshifted by 3 with clip after. Without this,
internal int16 accumulators in the decompiled effect functions overflow on
bright/transient material → harsh wraparound distortion.

LFO updates run at 1/8 the DSP rate (~2930 Hz at 23.4 kHz). LFO-driven
programs (some Midifex flangers) use upstream's LFO init which is not yet
wired up — a follow-on for ROM mode (Path B).

## Resampler

32-tap polyphase windowed-sinc FIR with 256 phases. Coefficients are
generated by `scripts/gen_resampler.py` and committed as a `static const`
table in `src/dsp/resampler_coeffs.h`.

The polyphase math: for output position p with frac = p - floor(p),
h[k] = window(k) * sinc(2 * cutoff * (k - center - frac)) for k = 0..L-1.
Critical to subtract frac, not add — `+ frac` reverses the phase ramp and
creates a signal-modulated buzz at `Fs / NUM_PHASES` and harmonics.

Both buffers are pre-filled with `RESAMPLER_TAPS` zeros at init so the FIR
window has lookahead from the very first call (otherwise the upsampler
underflows by ~32 samples per block and emits silence for the second half
of each block).

## Build

`./scripts/build.sh` (Docker + aarch64-linux-gnu-gcc) → `dist/midiverb-module.tar.gz`.

## Tests

`make -C tests/host all` — host-side smoke + resampler test.

The `tests/host/wav_runner` and `tests/host/mid_runner` binaries process
a WAV file through the full plugin or just the effect at 23.4 kHz, useful
for A/B comparison against `upstream/midiverb_emulator`'s CLI when
debugging audio quality.
