# Third-party licenses

## midiverb_emulator (vendored)

The following files are vendored from
https://github.com/thement/midiverb_emulator and are distributed under
the GPL-3.0-or-later (same as the rest of this module):

| File | Purpose | Modifications |
|------|---------|---------------|
| `decompiled-midiverb.h`  | Decompiled Midiverb I effect functions   | none |
| `decompiled-midifex.h`   | Decompiled Midifex effect functions      | none |
| `decompiled-midiverb2.h` | Decompiled Midiverb II effect functions  | none |
| `names-midiverb.h`       | Midiverb I preset names                  | none |
| `names-midifex.h`        | Midifex preset names                     | none |
| `names-midiverb2.h`      | Midiverb II preset names                 | none |
| `dasp16.h`               | DASP cycle-accurate interpreter (ROM mode) | functions made `static inline` so the header can be included in multiple translation units |
| `lfo.h`                  | LFO oscillator + per-program LFO patches | functions and `lfo_patches[]` made `static`/`static inline` for the same reason |
| `rom.h`                  | `RomType` table + reference loader       | unmodified — we only reference `rom_types[]`. We provide our own ROM loader (`mv_try_load_rom` in `midiverb_core.c`) because upstream's calls `die()` on failure, which would crash the audio host |
| `utils.h`                | Helpers (`ARRAY_SIZE`, `read_bytes`, `die`) | added explicit `<stdio.h>`/`<stdlib.h>`/`<string.h>`/`<errno.h>`/`<stdarg.h>` includes so the header can be used standalone. We do not use `read_bytes` or `die` |

## Resampler coefficient table

`resampler_coeffs.h` is auto-generated by `scripts/gen_resampler.py`
(also GPL-3.0-or-later in this repo). The Kaiser-windowed sinc math
follows standard polyphase FIR design — no third-party code reused.
